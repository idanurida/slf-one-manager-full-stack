# Project Rules & Constraints

## 1. Design & UI/UX Preservation (CRITICAL)
- **DO NOT CHANGE**: Layout, Design, Colors, or Themes.
- **Goal**: Maintain the existing visual identity exactly as is.
- **Action**: When fixing bugs or adding logic, touch ONLY the logic code. If a UI component needs to be used, reuse existing ones without modification.
- **Verification**: Ensure no visual regression occurs after any change.

## 2. Code Style
- Follow the existing project structure.
- Do not refactor simply for aesthetics unless requested.
